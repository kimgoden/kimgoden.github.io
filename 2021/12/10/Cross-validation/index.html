Cross_validation

교차검증

  • 테스트 데이터세트를 평가하기전, 학습과 검증세트에서 알고리즘 학습과 평가를 수행하는것
  • 데이터 편중을 막기 위해 별도의 여러 세트로 구성된 학습데이터 세트와 검증 데이터 세트에서 학습과 평가를 수행하는 것

K 폴드 교차 검증

  • 가장 보편적으로 사용되는 기법
  • k개의 데이터 폴드 세트를 만들어 k번 만큼 각 폴트 세트에 학습과 검증 평가를 반복적으로 수행하는 방법
  • 사이킷런에선 k폴드 교차 검즈을 위한 KFold, StratifiedKFold 클래스릉 제공한다. 
붓꽃 데이터 세트 크기: 150

  • split()을 호출해 전체 붓꽃 데이터를 5개의 폴드 데이터 세트로 분리
  • 총 150개의 데이터 세트가 120개의 데이터 세트, 30개의 검증 테스트 데이터 세트로 분리된다.

Stratified K 폴드

  • 불균형한 분포도를 가진 레이블(결정 클래스) 데이터 집합을 위한 폴드 방식이다.
  • 여기서 불균형한 분포도란, 특정 레이블 값이 특이하게 많거나 매우 적어 값의 분포가 한쪽으로 치우치는 것을 의미한다.
  • K폴드가 레이블 데이터 집합이 원본 데이터 집합의 레이블 분포를 학습 및 테스트 세트에 제대로 분배하지 못하는 경우 사용된다.
  • 이러한 특징으로 원본데이터의 레이블 분포를 먼저 고려한 뒤 이 분포와 동일하게 학습과 검증 데이터 세트를 분배합니다.
2    50
1    50
0    50
Name: label, dtype: int64

  • 교차검증시마다 3개의 폴드 세트로 만들어지는 학습 레이블과 검증 레이블이 완전히 다른 값으로 추출되었다.
  • 첫 번째 교차 검증의 경우 학습레이블의 1,2 각각 50개가 추출 되었고 검증 레이블의 0값이 50개 추출되었다.
  • 이런 경우 0 은 전혀 학습하지 못하게 된것이고, 반대로 1,2는 검증 레이블에 없기 때문에 예측이 불가능하다.
  • —> 이럴 경우 예측 정확도가 0이 나오게된다.

Stratified K 폴드 사용

  • 위와 같은 현상을 방지 하기위해 Stratifie K 폴드를 사용한다.
  • 사용방법은 KFold와 거의 비슷하다
  • 차이점은 레이블 데이터 분포도에 따라 학습/검증 데이터를 나누기 때문에 split()메서드에 인자로 피처 데이터 세트뿐만 아니라 레이블 데이터 세트도 반드시 필요하다.

교차검증 사이킷런 API

  • cross_val_score()

    1. 폴드 세트 설정
    2. for루프에서 반복으로 학습 및 테스트 데이터의 인덱스 추출
    3. 반복 학습/예측 수행 및 예측 성능 반환

  • cross_val_score()의 API 선언형태

  • cross_val_score(estimator. X. y=None, scoring=None, cv=None, n_jobs=1, verbose=0, fit_params=None, pre_dispatch='2*n_jobs')

  • 중요 파라미터: estimator. X. y=None, scoring=None, cv=None

  • estimator : 사이킷런의 분류/회귀 알고리즘 클래스를 의미

  • X : 피처 데이터 세트

  • y : 레이블 데이터 세트

  • scoring : 예측 성능 평가 지표를 기술

  • cv : 교차 검증 폴드 수

교차 검증별 정확도: [0.98 0.94 0.98]
평균 검증 정확도: 0.9667
  • cv에서 지정된 횟수만큼 scoring 파라미터로 지정된 평가 지표로 평과 결과값을 반환한다.
  • 비슷한 API로는 cross_validate()가 있다.
  • 차이점 : cross_val_score() 는 단 하나의 평가지표만 가능하지만, cross_validate()는 여러 개의 평가 지표를 반환할 수 있다.
  • 차이점2 : 학습 데이터에 대한 성능 평가 지표와 수행 시간도 같이 제공한다.
